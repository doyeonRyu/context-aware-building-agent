# 컨텍스트 인식형 건물 에너지 시뮬레이션 에이전트

> 2026.03.20. (금) 
> 스마트시티융합학과 석사 과정 유도연
---
## 서론
### 문제 제시
1. 전체 에너지 소비 중 건물 분야의 에너지 소비가 큼
2. 도시 건물 에너지와 주변 환경의 관계
    - 건물 에너지 성능은 건물 자체의 물리적 특성뿐만 아니라 주변 건물과의 상호작용(Inter-Building Effect, IBE)에 의해 크게 영향을 받음
    - Shading, Sky View Factor(SVF), 장파 복사 열교환 등 주변 건물로 인한 효과는 대상 건물의 냉난방 부하, 피크 수요, 월별 에너지 패턴을 상당 수준 변화시킴

### 선행 연구
1. AI Agent + 건물 에너지 분야는 여전히 단일 건물의 O&M, HVAC 제어 최적화에 초점
2. 대부분의 단일 건물 에너지 시뮬레이션(BES)는 주변 건물을 무시하거나 단순화하여 에너지 부하를 과소/과대 추정함
3. UMI, CitySim, CityEnergyAnalyst 등 기존 UBEM 도구들은 IBE를 자동으로 반영하지 않거나 전문가의 수동 설정을 요구
4. 기존 LLM-BEM 연구(Eplus-LLM 2024, LLM Agentic BEM 2025)는 가상 건물 또는 reference building 기반으로 실측 검증이 부재하며 주변 컨텍스트는 고려하지 않음
5. 온톨로지 기반 BEM 자동화(2023) 연구는 IBE와 LLM을 포함하지 않음
6. IBE가 Retrofit 우선순위 결정에 미치는 영향을 정량화한 연구 부재
    - context 미반영 시 잘못된 retrofit 의사결정 가능성 존재

- 선행 연구 대비 차별점
	
| **선행 연구**                                    | **한계**                     | **본 연구의 차별점**                   |
| -------------------------------------------- | -------------------------- | ------------------------------- |
| EPlus-LLM (2024) Applied Energy              | 가상 건물 기반, IBE 없음, 실측 검증 없음 | 실측 calibrated model + IBE 자동 통합 |
| LLM Agentic BEM (2025) Energy & Buildings    | 단일 건물 중심, 주변 환경 컨텍스트 없음    | 주변 건물 외형 기반 IBE 파이프라인 추가        |
| Ontology-based BEM (2023) Energy & Buildings | IBE 없음, LLM 없음             | 온톨로지 + IBE + LLM 동시 결합          |
| IBE 수동 연구들                                   | 수동 계산, 자동화 없음, 의사결정 분석 없음  | 자동화 파이프라인 + Decision Shift 분석   |
### 연구 공백
1. 실증 건물 + 주변 건물 외형 기반 IBE 자동 계산 + LLM 추론을 통합한 파이프라인이 부재
2. context 반영 여부가 retrofit 우선 순위와 기대 절감량 추정을 얼마나 바꾸는지 정량화한 연구 부재
3. 설계 단계에서 ‘앞의 건물이 생기면 에너지가 어떻게 달라지는가’를 자동 분석하는 end-to-end 시스템 부재

---
## 공헌
- 실측 에너지 데이터로 캘리브레이션된 대상 건물 모델과, 주변 건물 외형으로부터 자동 추출한 urban context를 연계하는 context-aware simulation pipeline을 구축하고, 
- 이 파이프라인을 통해 주변 건물 컨텍스트가 에너지 부하 및 retrofit 의사결정에 미치는 영향을 정량화

### Research Question
- RQ1
    - 주변 건물 컨텍스트 (높이, 거리, 방위, 차폐 등)이 대상 건물의 연간 에너지 소비, 냉난방 부하, 피크 수요, 월별 민감도에 어떤 영향을 미치는가?
- RQ2
    - 컨텍스트 반영 여부가 retrofit 대안의 우선순위와 기대 절감량 추정을 어떻게 바꾸는가? 
- RQ3
    - LLM Agent는 IBE 정보를 얼마나 정확하게 인식하여 컨텍스트 인식 추천을 생성하는가?

### 주요 공헌
1. Context-Aware Simulation Pipeline 제안
2. 다층적 에너지 영향 분석
3. 의사결정 영향 - Decision Shift 분석
4. 실증 - 명확히 제한된 범위의 정직한 검증

## 방법론

### 연구 범위

|**구분**|**범위**|**비고**|
|---|---|---|
|대상 건물|실제 건물 1동 (도면 + 에너지 사용량 보유)|실측 캘리브레이션 기준점|
|주변 건물|외형 정보만 (위치·높이·풋프린트)|Shading Surface 전용, 내부 정보 불필요|
|컨텍스트 변수|높이 / 이격거리 / 방위 / 복합 조합|파라메트릭 스터디|
|시뮬레이션 엔진|OpenStudio + EnergyPlus|Python API 자동화|
|LLM 역할|IBE 인식 → 시뮬레이션 설계 → 결과 해석 → 추천|에너지 수치 직접 생성 아님|
|검증 범위|대상 건물 base model만 실측 검증|주변 건물 실측 검증 제외 (한계로 명시)|

### 전체 파이프라인

| **단계**               | **입력**                              | **처리**                                            | **출력**                            |
| -------------------- | ----------------------------------- | ------------------------------------------------- | --------------------------------- |
| Step 1 Base Model 구축 | 실제 건물 도면 (BIM) 실측 에너지 사용량 기상 EPW 파일 | Revit → gbXML 변환 OpenStudio SDK 로드 표준 스케줄 적용      | Calibrated OSM 모델 CVRMSE ≤ 15% 확인 |
| Step 2 Context 입력    | 주변 건물 외형 (위치·높이·풋프린트) 자연어 시나리오 입력   | LLM Agent IBE 해석 Shading Surface 자동 생성 SVF·반사율 계산 | OpenStudio에 IBE 파라미터 자동 주입        |
| Step 3 시뮬레이션         | Context 반영 OSM EPW 기상 파일            | EnergyPlus 자동 실행 파라메트릭 스터디 다중 시나리오 병렬             | 냉난방 부하 연간 에너지 Peak load 월별 프로파일   |
| Step 4 해석·추천         | 시뮬레이션 결과 건물 온톨로지 지식                 | LLM 결과 해석 Context 전후 비교 Decision Shift 분석         | 자연어 추천 리포트 Retrofit 우선순위 절감량 추정   |

### 기술 스택

|**컴포넌트**|**도구**|**역할**|
|---|---|---|
|BIM 모델링|Revit → gbXML|대상 건물 정보 추출|
|주변 건물 모델링|SketchUp (시각화)|외형 확인·검수 (자동화는 Python)|
|에너지 모델 조작|OpenStudio Python SDK|gbXML 로드, Shading 주입, Measure 적용|
|IBE 계산|Python (기하 계산 / ladybug-core)|음영 마스크, SVF, 반사율 자동 산출|
|시뮬레이션 엔진|EnergyPlus Python API|물리 기반 에너지 계산, 결과 CSV 파싱|
|지식 표현|BOT/Brick 온톨로지, RDFLib|건물 정보 트리플, SPARQL 쿼리|
|LLM + RAG|LangChain + GraphRAG|온톨로지 기반 질의응답, 추천 생성|
|에이전트 오케스트레이션|LangGraph StateGraph|에이전트 간 흐름 관리, 피드백 루프|
|LLM 모델|GPT-4o / Claude Sonnet|IBE 해석, 추천 생성, 결과 설명|
### Agent의 Role과 Task

| Agent                | Role              | 핵심 Task                          | 부재시         |
| -------------------- | ----------------- | -------------------------------- | ----------- |
| **Context Agent**    | 자연어 → IBE 파라미터 변환 | 좌표 계산, Shading 생성, OSM 주입        | 수동 모델링 필요   |
| Simulation Agent     | 모델 자동 실행 + 결과 파싱  | 병렬 시뮬레이션, CSV 파싱, △에너지 계산        | 수동 실행 필요    |
| Analysis Agent       | 숫자 → 패턴·인과 해석     | 민감도 분석, 계절별 역전, 메커니즘 설명          | 결과가 숫자로만 남음 |
| Recommendation Agent | 지식 + 결과 → 의사결정 추천 | GraphRAG, Decision Shift, 절감량 추정 | 추천 없음       |

### 파라메트릭 시나리오 설계 (예상)
- 주변 건물 컨텍스트 조건별 에너지 영향을 정량화하기 위한 시뮬레이션 실험 설계

| **변수**   | **수준**                    | **시나리오 수** | **목적**            |
| -------- | ------------------------- | ---------- | ----------------- |
| 주변 건물 높이 | 없음 / 5m / 10m / 20m / 40m | 5          | 높이에 따른 음영 영향 크기   |
| 이격거리     | 3m / 6m / 10m / 15m       | 4          | 거리에 따른 음영 감쇠      |
| 차폐 방위    | N / S / E / W / 복합        | 5          | 방향별 영향 차이 및 계절 역전 |
| 실제 주변 건물 | 실제 단지 배치 반영               | 1          | 현실 조건 검증          |

## 결과

### 예상 결과물 형태
- 웹 서비스 형태
- 자연어 질의: 주변 컨텍스트 상황 설정
- Agent
	- 기존 base model에 주변 컨텍스트 자동 적용, 에너지 시뮬레이션
	- 기존의 base model 에너지 소비량과 시나리오 적용 시 에너지 소비량 비교 분석
### 실증
- 실증 대상 건물을 base model로 설정, 주변 컨텍스트를 직접 반영 시 vs agent 자동화 반영 시 결과 비교 
	- 수동 모델링 결과 대비 파이프라인 결과의 에너지 편차 ±X% 이내를 검증 기준으로 설정
	- 실증 대상 건물: BIM 정보, 에너지 소비량 등 데이터 존재
	- 주변 건물: 단순 외형 정보만 존재

2. 컨텍스트 반영 시 retrofit 시나리오를 통한 의사결정 제안
3. 추후 실내외 쾌적성  통합 / LLM 오케스트레이션 기반 도시 단위 에너지 교통 환경 통합 스마트시티 시스템으로 확장 가능
